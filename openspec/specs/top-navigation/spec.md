# Spec: 顶部导航栏展示与布局

## 概述

页面顶部提供固定导航栏：左侧为个人 Logo 或姓名；右侧为「首页」「项目」「关于我」「联系我」一级链接。不包含搜索、下拉菜单与登录注册。

## 行为规格

### 场景：导航栏固定在视口顶部

**Given** 用户已打开站点任意滚动位置  
**When** 用户查看页面顶部  
**Then** 导航栏始终固定在视口顶部可见区域内，且主内容不被导航栏永久遮挡（例如通过 `scroll-margin-top` 或主容器 `padding-top` 补偿）

### 场景：左侧品牌与右侧链接

**Given** 导航栏已渲染  
**When** 用户查看导航栏  
**Then** 左侧可见个人 Logo 或姓名（实现二选一或按配置展示其一，以 `design.md` 为准）；右侧可见四个链接：首页、项目、关于我、联系我，且顺序为从左到右：首页 → 项目 → 关于我 → 联系我

### 场景：点击品牌回到首屏

**Given** 用户已将页面向下滚动离开首屏 section  
**When** 用户点击左侧品牌区域（Logo 或姓名）  
**Then** 视口平滑滚动至首屏 section（与「锚点平滑滚动」spec 一致），且首屏 section 获得合理可见位置（不被导航遮挡）

## 边界与异常

### 场景：窄屏视口

**Given** 视口宽度小于常见手机横屏宽度  
**When** 用户查看导航栏  
**Then** 四个链接仍可访问（允许换行、压缩间距或横向轻滚动中的一种实现，以 `design.md` 锁定）；不得引入多级下拉菜单；不得引入搜索框

### 场景：键盘与焦点

**Given** 用户使用键盘 Tab 浏览  
**When** 焦点进入导航栏内链接  
**Then** 每个链接具备可见的 `:focus-visible` 样式；激活链接使用 Enter / Space 可触发与鼠标点击一致的导航滚动行为
