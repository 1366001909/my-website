# Spec: 滚动切换时的背景模糊反馈

## 概述

在用户通过主导航栏触发页面内锚点跳转时，在滚动过渡期间展示短时**背景模糊类**视觉反馈（例如全屏半透明层 + `backdrop-filter: blur(...)`），增强「正在切换 section」的感知；须遵守动效降级与可访问性约束。

## 行为规格

### 场景：点击导航后出现模糊反馈

**Given** 用户未开启 `prefers-reduced-motion: reduce`  
**When** 用户点击主导航中任一锚点链接（首页 / 项目 / 关于我 / 联系我）或左侧品牌触发滚动  
**Then** 在滚动开始至滚动结束（以 `scrollend` 事件或等效兜底定时器为准，见 `design.md`）期间，用户可见背景模糊反馈层；滚动结束后该层移除或透明度归零，不阻挡后续指针交互

### 场景：动效降级下关闭或减弱模糊

**Given** 用户系统为 `prefers-reduced-motion: reduce`  
**When** 用户触发同样的导航滚动  
**Then** 不展示全屏高模糊强度反馈，或仅展示极短、低强度的瞬时提示（以 `design.md` 二选一并实现）；不得强制长时长 `backdrop-filter` 动画

## 边界与异常

### 场景：快速连续点击多个导航链接

**Given** 用户在滚动尚未完成时再次点击另一导航链接  
**When** 第二次导航生效  
**Then** 模糊反馈与滚动目标切换不出现卡死、叠多层不可点击遮罩；最终仅保留一层可预测行为（取消上一次定时器或合并为单次过渡，以 `design.md` 为准）

### 场景：浏览器不支持 `backdrop-filter`

**Given** 运行环境不支持或关闭 `backdrop-filter`  
**When** 用户触发导航滚动  
**Then** 页面仍可完成锚点滚动；可提供降级（例如仅半透明遮罩无模糊），不得因检测失败而阻塞滚动
