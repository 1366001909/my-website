# Spec: 项目展示区块与卡片内容

## 概述

在 Hero Section 下方的 `#projects` 区域内，以卡片栅格展示至少 **4** 个个人项目。每张卡片包含：项目截图、项目名称、项目简介（短文案）、指向 GitHub 仓库的外链。截图须使用 `loading="lazy"`。

## 行为规格

### 场景：访客浏览项目区块

**Given** 用户已打开首页且页面已渲染至项目区块  
**When** 用户查看 `#projects` 区域  
**Then** 用户可见区域标题（与 `projects-heading` 关联）及至少 4 张项目卡片；每张卡片同时可见截图、名称、简介与可识别的 GitHub 链接入口

### 场景：点击 GitHub 链接

**Given** 某张项目卡片已展示  
**When** 用户点击该卡片上的 GitHub 链接  
**Then** 在新标签页打开对应 URL，且使用 `rel="noopener noreferrer"`（或与 `design.md` 等效的安全策略）

### 场景：响应式布局

**Given** 用户分别在桌面宽度与窄屏（如宽度小于 640px）下查看  
**When** 用户浏览项目区块  
**Then** 卡片栅格自动换行或单列堆叠，不出现横向溢出导致整页不可读；文案在卡片内可换行

## 边界与异常

### 场景：图片加载失败

**Given** 某项目截图 URL 无效或资源 404  
**When** 页面渲染该卡片  
**Then** 显示占位图或中性占位符，卡片其余字段仍可读；不得因单张图片失败导致整页崩溃

### 场景：GitHub 链接缺失或非法（数据层）

**Given** 某条项目数据缺少 `githubUrl` 或格式非法  
**When** 页面渲染  
**Then** 实现应跳过该条并在开发构建阶段报错，或在 `design.md` 约定下隐藏该卡片的链接并保留文案提示；**不得**在 Out of Scope 内实现搜索或详情页作为补救
