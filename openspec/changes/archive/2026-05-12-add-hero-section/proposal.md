# Proposal: add-hero-section

## Why

当前站点仍接近模板初始状态，缺少能够在首屏快速传达个人身份与价值的信息模块。  
新增 Hero Section 可在用户进入页面的第一时间展示姓名、职业与一句话介绍，强化个人品牌识别。  
同时补齐明亮/暗黑主题切换能力，使页面在不同阅读环境下保持一致的可读性与风格体验。

## What Changes

本次变更将新增并调整以下内容（仅限前端页面展示层）：

1. 新增全屏高度 Hero Section（`100svh`），并将核心文案在首屏居中展示：
   - Name（主标题）
   - Role（职业）
   - One-line intro（一句话介绍）
2. 新增 Hero 背景视觉分层：
   - 底层为静态 CSS gradient
   - 上层为静态 Canvas 粒子叠加（初始化与 resize 时重绘）
3. 新增主题切换能力：
   - 支持 light / dark 手动切换
   - 用户偏好本地持久化，并在后续访问中保持

## Out of Scope

以下内容明确不在本次变更范围内，严禁开发：

- 不做任何动画效果（包括 gradient 动画、粒子运动动画等）
- 不做导航栏（navbar）
- 不做后端 API

## Capabilities

### 1. Hero 首屏信息展示

提供全屏首屏区域，居中展示个人核心信息，确保在桌面端与移动端都能稳定阅读。

### 2. 静态科技感背景

提供由 CSS gradient 与 Canvas 粒子组成的静态背景层次，增强科技感视觉但不引入动画。

### 3. 主题切换与偏好记忆

支持 light / dark 主题切换，且在刷新与回访后保持用户上次选择。

## Impact

### 对现有功能的影响

- 将替换或重构当前模板化首屏内容，首页结构会有明显变化。
- 主题相关样式变量与页面根节点主题标记会新增或调整，可能影响全局配色继承逻辑。

### 性能与可访问性影响

- Canvas 引入轻量绘制开销，但因无持续动画，运行时成本可控。
- 需保障文字与背景对比度、主题切换按钮键盘可达与语义标签完整。

### 风险与边界

- 移动端首屏高度应使用 `100svh` 以降低浏览器地址栏造成的布局抖动风险。
- 高分屏场景下 Canvas 需按 `devicePixelRatio` 处理，避免粒子模糊。
- 若 JavaScript 不可用，页面应至少保留可读文案与 CSS gradient 背景的降级可用性。
