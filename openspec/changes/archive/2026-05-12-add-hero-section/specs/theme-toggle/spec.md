# Spec: 主题切换与偏好记忆

## 概述

站点支持 **light** 与 **dark** 两种主题。用户可通过界面控件手动切换；选择持久化到 `localStorage`，后续访问默认沿用用户选择。若用户从未保存过偏好，则跟随系统 `prefers-color-scheme`，直至用户首次手动切换。

## 行为规格

### 场景：首次访问且无已存偏好

**Given** 用户首次打开站点且 `localStorage` 中无有效主题偏好键  
**When** 页面完成首屏渲染  
**Then** 页面主题与当前系统 `prefers-color-scheme` 一致（light 或 dark）

### 场景：用户手动切换主题

**Given** 页面已展示主题切换控件  
**When** 用户点击切换为另一主题  
**Then** 页面立即应用对应主题样式（文本、背景、边框等与设计 token 一致），且切换控件状态与当前主题语义一致

### 场景：刷新后保持用户选择

**Given** 用户已通过控件选择某一主题且持久化成功  
**When** 用户刷新页面或再次在同域打开站点  
**Then** 页面以用户上次选择的主题展示，而非仅跟随系统主题

## 边界与异常

### 场景：`localStorage` 不可用或写入失败

**Given** 浏览器禁用存储或写入 `localStorage` 抛出异常  
**When** 用户切换主题  
**Then** 当前会话内主题仍可切换并正确展示；刷新后可能回退为系统主题或会话内最后一次成功状态（实现需明确一种一致行为，且不得导致页面崩溃）

### 场景：存储值损坏或非预期

**Given** `localStorage` 中存在非 `light`/`dark` 的非法值  
**When** 页面加载读取偏好  
**Then** 实现应忽略非法值并回退到系统 `prefers-color-scheme`（或文档约定的默认），且不阻塞首屏渲染
